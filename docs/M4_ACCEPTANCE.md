## 已形成的闭环

- 基础筏格、墙、板门、承重柱、楼梯、上层地板和斜顶进入统一建造选择；滚轮换件、`R` 旋转、`F` 切换合法层高，HUD 同步显示件型、方向、层高、成本和结构总数；
- 墙/门使用规范化共享边占位，地板/屋顶使用楼面占位，柱/楼梯使用格内占位；结构数、边界、重复占位和层高由领域层统一拒绝；
- 底层竖向结构必须有基础筏格，上层竖向结构必须有地板；地板/屋顶需要两点承重，墙、门、柱和楼梯落点可参与支撑；
- 拆除承重点前会验证全部后继结构，存在依赖时拒绝；鲨鱼咬毁基础筏格后按拓扑逐轮清除失去支撑的结构；
- 六类结构均由原创分件装配构成，包含交错木板、绑扎、钉件、门框/门扇、双梁七级楼梯和编叶斜顶；运行时按几何与材质合并为最多七个实例批次；
- 板门在世界中接受准星焦点与 `E` 开合，门框保持固定、门扇转动；关闭时阻挡玩家，打开后释放通道，并具有独立木铰/摩擦程序音层；
- 结构幽灵预览复用最终分件轮廓，合法、材料不足和结构无支撑使用不同状态；有效建造才扣材料和锤耐久，无效点击不产生隐藏成本；
- v14 保存结构稳定 ID、类型、格坐标、层高、方向、生命和开门状态；v15 继续保存经真实表面校验的筏上脚底层高，v14 玩家迁移到基础层，恶意悬空高度会吸附到同坐标真实表面或归零；
- 基础筏格、楼梯、上层地板与斜顶进入统一可行走表面采样；同一水平位置有多层时只选择当前脚底可跨越的最高层，不会从底层瞬移到楼板上；
- 四向楼梯按朝向从本格底部连续上升到相邻格下一层，视觉、承重落点、射线包围盒和行走方向一致；离开上层边缘会进入既有重力/入水链，上层跳跃只从上方落回真实表面；
- 墙、门、柱按玩家脚底层高过滤碰撞：底层玩家不会被上层墙阻挡，上层玩家会被对应上层结构阻挡；低矮生活/种植/研究设备不会隔着楼板阻挡玩家，帆桅和天线仍保留贯层碰撞；
- 楼板按最低承梁厚度、斜顶按连续屋脊坡面采样下表面；玩家上跳跨过底面时在碰撞帧截断垂直速度，保留水平惯性与既有落点状态，四向楼梯落点一侧留出明确入口，屋檐外不会产生假碰撞；
- 楼板与斜顶撞顶使用不同木质/纤维程序音层、少量向下尘屑和低幅镜头冲击；冻结诊断记录结构 ID、材质、世界高度与碰撞帧速度，避免浏览器门禁依赖瞬态截图；
- 六类结构按各自最大生命进入完整、受损、临界三档；实例颜色按木、深木、绳、金属和纤维分别变化，临界阶段以确定性轻微错位、下垂和缺口表达，但继续使用原七个实例批次；
- 深潮鲨只从外沿筏格选择可达结构，方向投影、层高和当前损伤共同参与确定性排序；受损目标必然优先，完整目标按概率与基础筏格竞争，结构归零会删除目标并按既有承重拓扑逐轮坍塌；
- 已进入受损阶段的外沿弱点会把载入后的首轮巡游从 34 秒缩短到 6.5 秒；每次结构/筏格撕咬都立即写入 v15，刷新不能清除伤害；
- 建造锤准星命中受损结构时进入专用修补态，HUD 显示结构名、生命、百分比和按材质定义的成本；扣料、生命增加、金色/材质碎屑、空间木裂/绳纤维音、锤耐久和保存只在实际修补成功后共同提交；
- 柱和楼梯的底层格位与设备占位共享冲突规则，避免同一空间出现不可解释的设备穿插；
- 512×320 低高度布局保持交互提示、建造面板和快捷栏各 10px 间距，最长结构成本不会改变面板尺寸或遮挡输入提示。

## 自动与浏览器证据

- `npm test -- --run`：39 个测试文件、231 项测试通过；新增 35 项结构装配、占位、支撑、可行走/头顶表面采样、撞顶速度截断、损伤阶段、外沿择靶、级联、修补钳制、分层碰撞、交互所有权、实例批次和 v15 清洗/迁移测试；
- `npm run build`：TypeScript 与生产构建通过；React 主入口、3D 世界和 Rapier 继续独立拆包；
- 最终构建 `building` 行为场景实际合上 `showcase-door`，随后滚轮选择木墙、旋转到东向并在 `(1,-1)` 放置；结构数 `10 -> 11`、漂木 `48 -> 45`、绳索 `24 -> 23`、锤耐久 `80 -> 79`，v15 同帧写入；
- 同一场景切换到二层后，在没有上层地板的 `(1,-1)` 再次点击，运行时返回 `invalid`，结构仍为 11、材料与锤耐久不变；
- 门与墙瞄准使用运行时局部相机/目标几何迭代产生真实 `mousemove`，不通过直接改相机或 Store 绕过交互；
- `building` 视觉场景在 512×320 通过面板边界和相交断言，并生成 `artifacts/screenshots/building-structures-desktop.png`；当前 Termux 的 1024×640 整页 readback 仍可能占用合成器，因此行为和视觉分段执行；
- `building` 多层场景真实按住 `W` 从约 `0.03m` 楼梯底连续升到 `2.18m` 地板，写入 v15 后用全新浏览器上下文冷启动恢复到上层；随后真实 `Space` 跳跃并落回地板，按住 `S` 下楼回到脚底 `0m`，全过程 Context 和固定步保持有效；
- `building ceiling` 最终构建分别从楼板和斜顶屋脊下真实按 `Space`：命中 `ceiling-floor` / `ceiling-roof`，冻结世界高度 `2.248m` / `2.543m`，垂直速度均为 `0`，随后回落基础层；多层场景最终 `ceilingHits=0`，证明楼梯入口和上层跳跃未被误封；
- `building` 攻防场景让深潮鲨真实两次命中外沿木墙，生命 `75 -> 41 -> 7`，同格基础保持 `100`，v15 每次同步写入；全新浏览器上下文恢复临界材质和 `7/110` 修补 HUD 后，三次真实锤击完成 `7 -> 51 -> 95 -> 110`，漂木 `12 -> 9`、锤耐久 `80 -> 77`；
- 攻击阶段在当前软件 GLES 使用 10 倍 `performance.now()` 时间源驱动正式固定步并继续受 `maxSubSteps=12` 约束；冷启动、修补和两张对照截图使用原生时间，因此该门禁只证明正确性，不计入目标 GPU 攻防节奏或性能通过。

人工连续检查：

1. 七种建造件在 30/60 FPS 下滚轮切换、旋转、换层和连续放置均只提交一次，预览不抖动、不穿过手中锤具；楼梯上下行、转向和上层跳跃没有卡阶或强烈镜头突变，楼板/斜顶撞顶不会穿层、连震或丢失水平惯性；
2. 门框、门扇、墙、柱、楼梯、地板与斜顶在平静海面和风暴倾斜时始终附着同一筏体坐标，不漂到岛屿或海面；
3. 开门、关门、建造、拒绝和拆除音色可辨，连续操作不会叠出削波或盖过鲨鱼预兆；
4. 1280×720 与 1920×1080 高低画质下结构实例批次、阴影、透明预览和 HUD 没有异常跳变；
5. 保存、刷新和迁移后结构、方向、生命、开门状态和承重链一致，恶意无支撑结构不会进入世界；
6. 鲨鱼直接撕咬六类暴露结构或基础时，目标、损伤阶段、级联结果、碎屑、空间声音、通知和保存发生在同一事务边界；锤修后的生命、材料和耐久与 HUD 一致。
